# The Deceptive Cache
Review this Java 17 code for a caching system. The developer wanted to create a specialized read-only view of a cache, but the system is throwing unexpected errors in production and experiencing weird data mutations. Find the bugs and design flaws.

```java
import java.util.HashMap;
import java.util.Map;

public class MemoryCache {
    protected Map<String, String> store = new HashMap<>();

    public void put(String key, String value) {
        store.put(key, value);
    }

    public String get(String key) {
        return store.get(key);
    }
    
    public Map<String, String> getRawStore() {
        return store;
    }
}

class ReadOnlyCache extends MemoryCache {
    @Override
    public void put(String key, String value) {
        throw new UnsupportedOperationException("Cannot write to read-only cache!");
    }
}
```

---ANSWER---

### Code Review Findings
1. **LSP Violation (Liskov Substitution Principle):** `ReadOnlyCache` alters the expected behavior of the base class by throwing an exception on a core method. Code expecting a `MemoryCache` will crash if given a `ReadOnlyCache`.
2. **Encapsulation Break:** `getRawStore()` returns a direct reference to the internal mutable `HashMap`. Any caller can clear or modify the map, completely bypassing the "read-only" restriction.
3. **Protected Field Abuse:** The `store` is `protected`, allowing subclasses or classes in the same package to directly mutate it, further breaking encapsulation.

### Mandatory Evaluation Points
1. Identifies the Liskov Substitution Principle (LSP) violation.
2. Points out the leaky abstraction in `getRawStore()` (returning mutable reference).
3. Suggests a fix (e.g., composition instead of inheritance, or an interface `ReadableCache` that `WritableCache` extends).

### Life Analogy
Imagine ordering a "functioning car" (MemoryCache) from a rental agency. When you get in and press the gas pedal (`put`), the car explodes (`UnsupportedOperationException`). The rental agency gave you a "Display Only Car" (`ReadOnlyCache`), violating your expectation of what a car should do.
