### What are Implicitly Declared Classes and Instance Main Methods?

Previewed in Java 21 (**JEP 445**) and refined in Java 22/23 (**JEP 463/477**), **Implicitly Declared Classes and Instance Main Methods** streamline the entry point for Java applications.

Historically, even the simplest "Hello World" application in Java required significant boilerplate that introduced advanced object-oriented concepts (classes, static access modifiers, String array parameters) before beginners could understand basic programming principles:

```java
// Traditional Java Hello World (Verbose)
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Java 21 simplifies this by allowing:
1. **Instance Main Methods**: `main` methods no longer need to be `public` or `static`, nor do they require `String[] args` parameters.
2. **Implicitly Declared Classes**: Files containing methods and fields outside of an explicit `class` declaration automatically belong to an implicitly declared top-level class.

---

### Flexibilities Introduced in Main Launcher Protocol

In Java 21+, the launcher prioritizes and selects main methods according to a simplified search hierarchy:

1. `void main(String[] args)` (instance or static)
2. `void main()` (instance or static)

The access modifier can be `public`, `protected`, or default package-private (`private` main methods are not launched).

#### Valid Java 21 Main Entry Points

```java
// 1. Instance main with no parameters
void main() {
    System.out.println("Hello from instance main!");
}

// 2. Package-private static main without args
static void main() {
    System.out.println("Hello from static main!");
}

// 3. Instance main with args
void main(String[] args) {
    System.out.println("Args count: " + args.length);
}
```

When an instance `main` method is selected, the JVM automatically invokes a parameterless constructor of the class to instantiate it before executing `main()`.

---

### Implicitly Declared Classes Syntax

An **implicitly declared class** is created simply by writing fields and methods directly in a `.java` file without surrounding them in a `class Name { ... }` block:

```java
// File: SimpleApp.java (No explicit class header!)

String greeting = "Hello, Modern Java 21!";

void main() {
    printMessage(greeting);
}

void printMessage(String msg) {
    System.out.println(msg);
}
```

#### Under the Hood Mechanics

When compiling `SimpleApp.java`, the Java compiler automatically generates an implicit class named after the source file (e.g., `SimpleApp.class` in the unnamed package). Fields like `greeting` become member fields of this implicit class, and `printMessage` becomes an instance method.

### Comparison Matrix

| Feature | Legacy Java (<= 17) | Modern Java 21+ |
| :--- | :--- | :--- |
| **Class Declaration** | Mandatory (`public class Foo`) | Optional (Implicit top-level class) |
| **Main Modifier** | Must be `public static` | Can be package-private / instance |
| **Main Parameters** | Mandatory `String[] args` | Optional (`void main()`) |
| **Package** | Explicit or unnamed | Unnamed package for implicit class |
| **Instantiation** | Static execution only | Automatic default constructor invocation |

### Execution with Single-File Source Code Launcher

Combined with JEP 330 (Single-File Source-Code Execution), running small utilities or scripts becomes as simple as executing a script:

```bash
# Run Java 21 preview source file directly
java --enable-preview SimpleApp.java
```

### Practical Use Cases

- **Prototyping & Scripting**: Writing quick CLI utilities or data transformation scripts without boilerplate class setup.
- **Educational Accessibility**: Allows new programmers to write imperative statements immediately without needing to explain OOP inheritance, class structures, or static modifiers on day one.
- **Microservice Utilities**: Writing simple single-purpose tools cleanly.
