# Implicitly Declared Classes & Main Methods
Java 21 (Preview) introduces Implicitly Declared Classes and Instance Main Methods. How do these features lower the barrier to entry for beginners and reduce boilerplate for simple scripts?

---ANSWER---

For decades, Java has been criticized for its extreme verbosity, especially for beginners writing their very first "Hello, World!" program. A beginner must understand classes, access modifiers (`public`), static contexts (`static`), and array parameters (`String[] args`) before they can print a single line of text. This steep learning curve is intimidating compared to scripting languages like Python or JavaScript, where a single line of code suffices. Furthermore, writing quick, single-file utilities in Java always required scaffolding this heavy boilerplate.

Java 21 introduces (as a preview feature) a massively simplified entry point model designed specifically to solve this problem. 

**Instance Main Methods:** You no longer need `public static void main(String[] args)`. You can now define a non-static `void main()` method with no arguments. The JVM will automatically instantiate the class and call this method.

**Implicitly Declared Classes:** You don't even need to declare a `class` block anymore. If a `.java` file contains a `main` method (or simply top-level methods and fields) without an enclosing `class` declaration, the Java compiler implicitly wraps the entire file in an anonymous class. 

Together, these features mean you can write a fully functioning Java program in a single line of code, without any class declarations, `public`, `static`, or array arguments. It makes Java highly competitive for quick scripting and drastically improves the teaching experience for newcomers.

### Examples
```java
// Pre-Java 21: The notorious boilerplate
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

// Java 21 (Preview): Instance main method (Still needs a class)
class ModernHello {
    void main() {
        System.out.println("Hello, World!");
    }
}

// Java 21 (Preview): Implicitly Declared Class
// The ENTIRE file consists of just these three lines!
void main() {
    System.out.println("Hello, World!");
}
```

### Life Analogy
The old way is like requiring a customer to incorporate a legal business entity (a Class), appoint a CEO (a static method), and hire a receptionist to handle arbitrary packages (`String[] args`) just to buy a cup of coffee.

The new way is like a customer just walking up to the counter and saying "Coffee, please" (`void main()`). The system handles all the underlying mechanics invisibly, letting you focus entirely on the actual task you want to accomplish.

### Key Points
- Reduces the boilerplate required to launch a Java application.
- Introduces `void main()` (no `static`, no `public`, no `String[] args`).
- Allows writing `.java` files without an enclosing `class` declaration (Implicitly Declared Classes).
- Primarily designed to help students learn Java faster and to make writing small scripts much easier.
