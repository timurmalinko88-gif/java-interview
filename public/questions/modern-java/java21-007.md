### What is the Sequenced Collections API in Java 21?

Introduced in Java 21 via **JEP 431**, the **Sequenced Collections API** addresses a long-standing inconsistency in the Java Collections Framework (JCF): the absence of a unified interface representing collections with a defined encounter order.

Prior to Java 21, collections like `List`, `Deque`, `SortedSet`, and `LinkedHashSet` all maintained element order, but accessing the first or last element, or iterating in reverse order, required completely different, non-uniform API calls:

| Collection Type | First Element | Last Element | Reverse Iteration |
| :--- | :--- | :--- | :--- |
| `List` | `list.get(0)` | `list.get(list.size() - 1)` | `list.listIterator(list.size())` |
| `Deque` | `deque.getFirst()` | `deque.getLast()` | `deque.descendingIterator()` |
| `SortedSet` | `set.first()` | `set.last()` | `new TreeSet<>(set).descendingSet()` |
| `LinkedHashSet` | `set.iterator().next()` | Requires iterating to end | Not supported directly |

JEP 431 retrofitted the Collections Framework with three core interfaces:
1. `SequencedCollection<E>`
2. `SequencedSet<E>`
3. `SequencedMap<K, V>`

---

### Interface Hierarchy & Core Methods

#### 1. `SequencedCollection<E>`

Extends `Collection<E>` and represents a collection whose elements have a defined first, second, and N-th element up to the last element.

```java
public interface SequencedCollection<E> extends Collection<E> {
    // Reverse view
    SequencedCollection<E> reversed();

    // Access methods
    void addFirst(E element);
    void addLast(E element);
    E getFirst();
    E getLast();
    E removeFirst();
    E removeLast();
}
```

#### 2. `SequencedSet<E>`

Extends `SequencedCollection<E>` and `Set<E>`. Implemented by `SortedSet` (e.g., `TreeSet`) and `LinkedHashSet`.

```java
public interface SequencedSet<E> extends SequencedCollection<E>, Set<E> {
    SequencedSet<E> reversed();
}
```

#### 3. `SequencedMap<K, V>`

Represents a `Map` whose entries have a defined encounter order. Implemented by `SortedMap` (e.g., `TreeMap`) and `LinkedHashMap`.

```java
public interface SequencedMap<K, V> extends Map<K, V> {
    SequencedMap<K, V> reversed();

    SequencedSet<K> sequencedKeySet();
    SequencedCollection<V> sequencedValues();
    SequencedSet<Entry<K, V>> sequencedEntrySet();

    Entry<K, V> firstEntry();
    Entry<K, V> lastEntry();
    Entry<K, V> pollFirstEntry();
    Entry<K, V> pollLastEntry();
    V putFirst(K key, V value);
    V putLast(K key, V value);
}
```

### Code Examples

#### Standardized Operations across Collections

```java
import java.util.*;

public class SequencedCollectionDemo {
    public static void main(String[] args) {
        SequencedCollection<String> list = new ArrayList<>(List.of("A", "B", "C"));
        
        System.out.println(list.getFirst()); // "A"
        System.out.println(list.getLast());  // "C"

        list.addFirst("START");
        list.addLast("END");

        System.out.println(list); // [START, A, B, C, END]
        System.out.println(list.reversed()); // [END, C, B, A, START]
    }
}
```

#### LinkedHashSet Support

Prior to Java 21, accessing the last element of a `LinkedHashSet` required an $O(N)$ iteration. Now it is $O(1)$:

```java
SequencedSet<String> set = new LinkedHashSet<>();
set.add("First");
set.add("Second");
set.add("Third");

String last = set.getLast(); // "Third" - O(1) complexity
System.out.println("Reversed set: " + set.reversed()); // [Third, Second, First]
```

#### SequencedMap Operations

```java
SequencedMap<String, Integer> map = new LinkedHashMap<>();
map.put("One", 1);
map.put("Two", 2);
map.putFirst("Zero", 0);

System.out.println(map.firstEntry()); // Zero=0
System.out.println(map.lastEntry());  // Two=2

// Reversing map view without copying memory
SequencedMap<String, Integer> reversedMap = map.reversed();
System.out.println(reversedMap); // {Two=2, One=1, Zero=0}
```

### Key Behavior: The `reversed()` View

The `reversed()` method returns a **live view** of the original collection in reverse order, not a copy. Modifications to the reversed view mutate the underlying collection:

```java
SequencedCollection<String> original = new ArrayList<>(List.of("1", "2", "3"));
SequencedCollection<String> rev = original.reversed();

rev.addFirst("4"); // Adds to the END of original!
System.out.println(original); // [1, 2, 3, 4]
```

---

### Summary of Benefits

- **Uniformity**: Standardized API for common operations across `List`, `Set`, and `Map`.
- **Efficiency**: Direct $O(1)$ access to first and last elements in `LinkedHashSet` and `LinkedHashMap`.
- **Zero-allocation views**: `reversed()` creates dynamic views without duplicating backing arrays or nodes.
