### What are Record Patterns in Java 21?

Java 21 finalized **Record Patterns** (JEP 440), expanding the pattern-matching capabilities introduced in earlier versions for `instanceof` and `switch` statements. Record patterns enable developers to deconstruct record instances into their component values in a concise, type-safe, and declarative manner without requiring manual getter calls.

Before Java 21, extracting components from nested record structures required casting and invoking accessors repeatedly:

```java
// Prior to Java 21
if (obj instanceof Point p) {
    int x = p.x();
    int y = p.y();
    // use x and y
}
```

With Record Patterns in Java 21, pattern matching decomposes the record directly at the site of test:

```java
// Java 21 Record Pattern
if (obj instanceof Point(int x, int y)) {
    System.out.println("X: " + x + ", Y: " + y);
}
```

### Nested Record Destructuring

The true power of record patterns shines when handling complex, deeply nested data structures. Record patterns support **nested destructuring**, allowing multi-level record hierarchies to be unwrapped in a single match expression.

#### Example Domain Model

```java
public record Point(int x, int y) {}
public record Color(int r, int g, int b) {}
public record ColoredPoint(Point point, Color color) {}
public record Window(ColoredPoint topLeft, ColoredPoint bottomRight) {}
```

#### Multi-Level Extraction

Without record patterns, extracting the red component of the top-left corner of a window requires verbose chain navigation and null checks:

```java
// Legacy Approach
if (w != null && w.topLeft() != null && w.topLeft().color() != null) {
    int red = w.topLeft().color().r();
    // proceed
}
```

Using nested record patterns in Java 21:

```java
public static void printTopLeftRed(Window window) {
    if (window instanceof Window(
            ColoredPoint(Point(int x, int y), Color(int r, int g, int b)),
            ColoredPoint bottomRight
    )) {
        System.out.println("Top-left red intensity: " + r + " at location (" + x + ", " + y + ")");
    }
}
```

In this single pattern expression, Java automatically verifies object structure, extracts nested values (`x`, `y`, `r`, `g`, `b`), and binds them to local scope variables safely.

### Record Patterns in Switch Expressions

Record patterns integrate seamlessly into `switch` expressions and statements. Combining record patterns with sealed types provides compile-time exhaustiveness checking.

```java
public sealed interface Shape permits Circle, Rectangle {}
public record Circle(Point center, double radius) implements Shape {}
public record Rectangle(Point topLeft, Point bottomRight) implements Shape {}

public static double calculateArea(Shape shape) {
    return switch (shape) {
        case Circle(Point center, double radius) -> Math.PI * radius * radius;
        case Rectangle(Point(int x1, int y1), Point(int x2, int y2)) -> {
            int width = Math.abs(x2 - x1);
            int height = Math.abs(y2 - y1);
            yield width * height;
        }
    };
}
```

Notice how `Rectangle` destructures its nested `Point` record components directly inside the `switch` arm. If new subtypes are added to `Shape`, the compiler will report an unhandled case error.

### Type Inference with `var` in Record Patterns

Java 21 allows type inference using `var` inside record patterns to keep code clean when types are lengthy or obvious:

```java
if (obj instanceof Window(ColoredPoint(Point(var x, var y), var color), var br)) {
    System.out.println("Point coordinates: " + x + ", " + y);
}
```

### Key Architectural Benefits

1. **Boilerplate Elimination**: Eliminates explicit accessor calls (`.x()`, `.color()`) during structural pattern matching.
2. **Improved Readability**: Declarative data extraction mirrors the record declaration syntax.
3. **Type Safety & Exhaustiveness**: Works with sealed class hierarchies to ensure all structural shapes are handled safely at compile-time.
4. **Immutability Synergy**: Pairs naturally with immutable records to create expressive data-oriented programming pipelines in modern Java.
