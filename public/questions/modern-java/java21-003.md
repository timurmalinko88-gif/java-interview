### Structured Concurrency in Java 21

Structured Concurrency is a preview feature in Java 21 (introduced as part of Project Loom) that aims to simplify multithreaded programming by treating multiple concurrent tasks running in different threads as a single unit of work. It is deeply integrated with Virtual Threads and relies on the new `StructuredTaskScope` API.

#### The Problem with Unstructured Concurrency
Before Structured Concurrency, Java developers typically used `ExecutorService` or `CompletableFuture` to spawn parallel tasks. This approach is "unstructured" because the lifecycle of the spawned threads is not tightly bound to the code block that created them. 

Consider a method that fetches user details and user orders concurrently using an `ExecutorService`. If the order-fetching task fails and throws an exception, the user-details task continues running in the background, wasting resources. Worse, if the main thread gets interrupted or cancelled, the spawned background threads might not be cancelled properly, leading to thread leaks or orphaned tasks (sometimes called "zombie" threads). Keeping track of errors, cancellation, and joining across multiple independent threads is notoriously error-prone.

#### The Solution: Structured Concurrency
Structured Concurrency solves this by ensuring that if a task splits into concurrent subtasks, they all return to the same place (the scope block). The lifetime of the subtasks is strictly confined to the lexical scope in which they were created. 

If any subtask fails, or if the main task is interrupted, all other sibling subtasks in the same scope can be automatically cancelled. This creates a clear hierarchy of tasks, much like a call stack for threads, making concurrent code much easier to reason about, debug, and maintain.

#### `StructuredTaskScope`
The core class for structured concurrency is `StructuredTaskScope`. It implements `AutoCloseable`, allowing it to be used in a `try-with-resources` block. The `try-with-resources` block ensures that the execution does not exit the block until all submitted subtasks have completed (either successfully or exceptionally).

Java provides two common built-in policies:
1. `StructuredTaskScope.ShutdownOnFailure`: Cancels all other tasks if any task fails. Used for "all-or-nothing" scenarios (e.g., gathering multiple independent pieces of data to construct a single response).
2. `StructuredTaskScope.ShutdownOnSuccess`: Cancels all other tasks as soon as any one task succeeds. Used for "first-to-finish" scenarios (e.g., querying multiple redundant servers and taking the first successful response).

#### Code Example

```java
import java.util.concurrent.StructuredTaskScope;
import java.util.concurrent.ExecutionException;
import java.util.function.Supplier;

public class StructuredConcurrencyExample {

    record UserProfile(String name) {}
    record OrderHistory(int orderCount) {}
    record UserDashboard(UserProfile profile, OrderHistory history) {}

    // Simulated blocking I/O calls
    private UserProfile fetchUserProfile(String userId) throws InterruptedException {
        Thread.sleep(500); // Simulate network delay
        return new UserProfile("Alice");
    }

    private OrderHistory fetchOrderHistory(String userId) throws InterruptedException {
        Thread.sleep(1000); // Simulate network delay
        // Throwing an exception here would cause the scope to shut down automatically!
        // throw new RuntimeException("Service unavailable");
        return new OrderHistory(42);
    }

    public UserDashboard getDashboard(String userId) throws InterruptedException, ExecutionException {
        // Create a scope that shuts down if ANY subtask fails.
        // It runs tasks using Virtual Threads by default.
        try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
            
            // Fork independent subtasks concurrently
            StructuredTaskScope.Subtask<UserProfile> profileTask = scope.fork(() -> fetchUserProfile(userId));
            StructuredTaskScope.Subtask<OrderHistory> historyTask = scope.fork(() -> fetchOrderHistory(userId));

            // Wait for all subtasks to finish (or for one to fail)
            scope.join(); 
            
            // Propagate any exceptions thrown by subtasks
            scope.throwIfFailed(); 

            // If we reach here, all subtasks succeeded. 
            // We can safely call .get() on the Subtask objects.
            return new UserDashboard(profileTask.get(), historyTask.get());
        }
    }

    public static void main(String[] args) throws Exception {
        var example = new StructuredConcurrencyExample();
        System.out.println(example.getDashboard("user-123"));
    }
}
```

#### Key Benefits
- **No Orphaned Threads:** The `try-with-resources` block guarantees that execution does not leave the scope until all threads have terminated.
- **Short-circuiting:** Using policies like `ShutdownOnFailure`, if one task throws an exception, the remaining tasks are immediately interrupted and cancelled, saving CPU and network resources.
- **Clear Error Propagation:** Exceptions are marshalled neatly back to the parent thread instead of being lost in background executors.

By unifying the lifecycle of concurrent tasks, Structured Concurrency brings the clarity and predictability of sequential programming to parallel execution in Java.
