# String Templates & Text Blocks
Explain how String Templates (Preview in Java 21) and Text Blocks improve string manipulation in Java. How are String Templates safer than traditional string concatenation or `String.format()`?

---ANSWER---

For years, constructing strings dynamically in Java was tedious and error-prone. You had to use the `+` operator (which was hard to read), `StringBuilder` (verbose), or `String.format()` (which was brittle, as a mismatch between the format specifier `%s` and the passed argument would cause a runtime crash). Furthermore, writing multi-line strings (like JSON, HTML, or SQL queries) required escaping newlines `\n` and quotes `\"` manually.

**Text Blocks** (introduced earlier, but widely adopted alongside Java 21) solved the multi-line issue. By wrapping text in triple quotes (`"""`), developers can write multi-line strings natively. Text blocks automatically handle internal quotation marks without escaping and smartly strip incidental leading whitespace, preserving the intended indentation of the code.

**String Templates**, introduced as a preview feature in Java 21, solve the dynamic construction problem. They allow developers to embed expressions directly inside a string literal using the `\{expression}` syntax. 

Crucially, String Templates in Java are inherently safer than simple string interpolation found in other languages (like JavaScript or Python). Java requires a **Template Processor** (like the built-in `STR` processor) to evaluate the template. This processor takes the literal text and the embedded values and constructs the final result. 
This design prevents massive security vulnerabilities like SQL Injection or Cross-Site Scripting (XSS). For example, a custom `SQL` template processor could automatically sanitize the embedded variables before constructing the final SQL query, making injection attacks impossible at the language level.

*(Note: String Templates were a preview feature in Java 21 and were surprisingly removed in Java 23 for a complete redesign, but understanding the Java 21 implementation is still commonly tested).*

### Examples
```java
String name = "Joan";
String phone = "555-1234";

// Pre-Java 21: Clunky concatenation
String jsonOld = "{\n" +
                 "  \"name\": \"" + name + "\",\n" +
                 "  \"phone\": \"" + phone + "\"\n" +
                 "}";

// Java 21: Text Blocks + String Templates (Preview)
// The STR processor replaces \{name} with the variable value.
String jsonModern = STR."""
    {
      "name": "\{name}",
      "phone": "\{phone}"
    }
    """;
```

### Life Analogy
Traditional string concatenation is like cutting out individual words from a magazine and gluing them onto a piece of paper to write a letter. It's messy, slow, and hard to read. `String.format` is like a Mad Libs puzzle; you have blanks, but if you accidentally put a number where a noun should go, the whole sentence makes no sense.

String Templates with Text Blocks are like having a smart, fill-in-the-blank digital form. You type your text naturally on multiple lines, drop the exact variable exactly where it belongs, and the Template Processor (the form software) ensures that if you drop in a dangerous script, it gets neutralized before the final document is printed.

### Key Points
- Text Blocks (`"""`) allow for easy creation of multi-line strings without manually escaping newlines or quotes.
- String Templates (Preview in 21) allow embedding expressions `\{var}` directly inside strings.
- They require a Template Processor (like `STR`) which safely combines the text and the variables.
- This processor-based design allows for custom validation and sanitization, preventing SQL injection and XSS.
