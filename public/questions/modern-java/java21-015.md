### What are String Templates in Java 21?

Previewed in Java 21 via **JEP 430**, **String Templates** bring safe, type-aware string interpolation to Java.

Before Java 21, constructing dynamic strings required concatenation (`+`), `String.format()`, `StringBuilder`, or `MessageFormat`. These approaches suffered from readability issues, missing compile-time checks, and critical security vulnerabilities like **SQL Injection** or **Cross-Site Scripting (XSS)** when handling untrusted user input.

String Templates solve these issues by combining literal text with embedded expressions `\{expression}`, evaluated safely through a **Template Processor**.

---

### Mechanics of Template Expressions

A String Template expression consists of three parts:
1. **Template Processor** (e.g., `STR`, `FMT`, or custom processors): Interpolates and transforms the template.
2. **Template String**: Contains embedded variables or expressions enclosed in `\{ ... }`.
3. **Embedded Expressions**: Any valid Java expression evaluated at runtime.

```java
String name = "Alice";
int age = 30;

// STR template processor
String message = STR."User \{name} is \{age} years old.";
System.out.println(message); // "User Alice is 30 years old."
```

#### Supported Template Processors

- **`STR`**: Performs simple string interpolation, converting expressions into strings.
- **`FMT`**: Performs interpolation while applying `java.util.Formatter` specifiers (e.g., decimal padding, currency alignment).
- **`RAW`**: Returns a raw `StringTemplate` object without processing, allowing custom processing.

```java
double price = 49.995;
int quantity = 3;

// FMT template processor with format specifiers
String receipt = FMT."Total: %-10s[name] | Price: %7.2f[price] | Qty: %03d[quantity]";
```

### String Templates with Multi-line Text Blocks

String Templates pair naturally with Java 15 **Multi-line Text Blocks** (`""" ... """`) to generate structured formats like JSON, HTML, XML, or SQL queries cleanly:

#### 1. Generating JSON Payloads

```java
public static String createJsonUser(String id, String name, String email, boolean active) {
    return STR."""
        {
            "user": {
                "id": "\{id}",
                "name": "\{name}",
                "email": "\{email}",
                "status": "\{active ? "ACTIVE" : "INACTIVE"}"
            }
        }
        """;
}
```

#### 2. Generating HTML Documents

```java
public static String renderHtmlPage(String title, List<String> items) {
    return STR."""
        <!DOCTYPE html>
        <html>
        <head><title>\{title}</title></head>
        <body>
            <h1>\{title}</h1>
            <ul>
                \{items.stream().map(i -> "<li>" + i + "</li>").collect(Collectors.joining("\n        "))}
            </ul>
        </body>
        </html>
        """;
}
```

### Custom Template Processors & Security (Preventing SQL Injection)

Unlike simple string interpolation in other languages (which yields plain strings directly), Java String Templates produce a `StringTemplate` object passed to a processor. This allows developers to write custom processors that validate or sanitize inputs before string creation.

#### Custom Safe SQL Query Processor Example

```java
import java.lang.StringTemplate;

public class SqlTemplate {
    // Custom processor producing PreparedStatement parameter bindings instead of concatenated strings
    public static final StringTemplate.Processor<PreparedStatement, SQLException> SQL = template -> {
        String sql = String.join("?", template.fragments());
        Connection conn = DatabaseConnection.get();
        PreparedStatement stmt = conn.prepareStatement(sql);
        
        List<Object> values = template.values();
        for (int i = 0; i < values.size(); i++) {
            stmt.setObject(i + 1, values.get(i));
        }
        return stmt;
    };
}

// Usage: Completely safe against SQL Injection!
String userInput = "admin' OR '1'='1";
PreparedStatement stmt = SqlTemplate.SQL."SELECT * FROM users WHERE username = \{userInput}";
```

---

### Summary of Benefits

1. **Readability**: Replaces unwieldy concatenation (`"..." + x + "..."`) and positional specifiers (`%s`) with inline expressions `\{x}` inside multi-line text blocks.
2. **Type Safety & Validation**: Embedded expressions are validated at compile-time.
3. **Security by Design**: Custom processors enforce domain-level sanitization (SQL, HTML escaping) rather than blindly concatenating raw input strings.
