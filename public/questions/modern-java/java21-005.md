### Pattern Matching for Switch in Java 21

Pattern Matching for `switch` is finalized in Java 21, fundamentally changing how developers write conditional logic in Java. It allows a `switch` expression or statement to test a variable against multiple types and patterns, extract values safely, and handle logic in a more readable, declarative manner compared to long chains of `if-else` and `instanceof` checks.

#### The Problem Before Java 21
Prior to this feature, `switch` was highly restricted. It could only evaluate primitive integers, enums, and Strings. If you received an object of type `Object` and needed to act based on its actual subclass, you were forced to write tedious `if-else` chains with explicit casting:

```java
// Pre-Java 21 Approach
void process(Object obj) {
    if (obj instanceof String s) {
        System.out.println("String of length: " + s.length());
    } else if (obj instanceof Integer i) {
        System.out.println("Integer value: " + i);
    } else if (obj == null) {
        System.out.println("Got null!");
    } else {
        System.out.println("Unknown type");
    }
}
```

#### The Solution: Pattern Matching in Switch
Java 21 allows you to use type patterns directly as `case` labels in a `switch`. The compiler automatically casts the variable, and the syntax is much cleaner. Additionally, Java 21 introduces **Guarded Patterns** (using the `when` clause) and enforces **Exhaustiveness** for sealed hierarchies.

#### Key Features
1. **Type Patterns:** You can match based on the type (`case String s`).
2. **Guarded Patterns (`when` clause):** You can add boolean conditions directly to a `case` label.
3. **Null Handling:** You can now explicitly handle null inside the `switch` using `case null`, removing the need for a pre-switch null check.
4. **Exhaustiveness:** If you are switching over a `sealed` class or an `enum`, the compiler guarantees that you have covered all possible subtypes. If you add a new subtype later, the compiler will fail, preventing bugs.

#### Code Example

```java
public class SwitchPatternMatching {

    // A sealed interface with specific implementations
    sealed interface Shape permits Circle, Rectangle, Triangle {}
    record Circle(double radius) implements Shape {}
    record Rectangle(double width, double height) implements Shape {}
    record Triangle(double base, double height) implements Shape {}

    // Method demonstrating Pattern Matching and Exhaustiveness
    public static double calculateArea(Shape shape) {
        // Because Shape is sealed, the compiler knows all possible subclasses.
        // No "default" branch is needed! If we add a Square class later,
        // the compiler will force us to update this switch.
        return switch (shape) {
            case Circle c -> Math.PI * c.radius() * c.radius();
            case Rectangle r -> r.width() * r.height();
            case Triangle t -> (t.base() * t.height()) / 2;
            // No default required for exhaustive switches over sealed types
        };
    }

    // Method demonstrating Guarded Patterns and null handling
    public static String formatData(Object data) {
        return switch (data) {
            case null -> "Data is missing";
            
            // Guarded pattern using the 'when' keyword
            case String s when s.isBlank() -> "Empty String";
            case String s when s.length() > 50 -> "Long String: " + s.substring(0, 50) + "...";
            case String s -> "Short String: " + s;
            
            case Integer i when i < 0 -> "Negative Number";
            case Integer i -> "Positive Number: " + i;
            
            // Default is required when switching on Object because it is not exhaustive
            default -> "Unsupported type: " + data.getClass().getSimpleName();
        };
    }

    public static void main(String[] args) {
        System.out.println(calculateArea(new Circle(5)));
        System.out.println(formatData("Hello Java 21"));
        System.out.println(formatData(null));
    }
}
```

#### Summary
Pattern Matching for `switch` transforms the switch construct from a simple equality-checker into a powerful type-checking and data-extraction tool. With the addition of `case null`, guarded patterns using `when`, and compiler-enforced exhaustiveness for sealed types, it makes Java code significantly more robust, readable, and safer against future refactoring.
