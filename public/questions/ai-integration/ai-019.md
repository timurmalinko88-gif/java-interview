# Spring AI auto-configurations

The power of the Spring Boot framework lies in its "convention over configuration" philosophy, driven by auto-configurations. **Spring AI** deeply embraces this philosophy. It leverages Spring Boot's autoconfiguration mechanics to allow developers to integrate complex AI models and vector databases simply by adding dependencies and providing API keys in the `application.properties` file.

### How Auto-Configuration Works in Spring AI

When you include a Spring AI starter dependency (like `spring-ai-openai-spring-boot-starter`), Spring Boot's autoconfiguration engine scans the classpath. Because it sees the OpenAI libraries, it automatically creates and registers several critical Beans in the application context without you writing a single line of `@Bean` configuration code.

The most important beans automatically created are:
1. `ChatClient.Builder`: A fluent builder to create customizable `ChatClient` instances.
2. `EmbeddingModel`: Used for converting text to vectors.
3. `ImageModel`: For image generation capabilities.

### Example: Zero-Code Configuration

#### 1. The POM Dependency
Add the starter to your `pom.xml`:

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
</dependency>
```

#### 2. The Properties
Provide your credentials and desired model parameters in `application.properties`:

```properties
spring.ai.openai.api-key=${OPENAI_API_KEY}
# Optional: Change the default model
spring.ai.openai.chat.options.model=gpt-4o
# Optional: Adjust the temperature
spring.ai.openai.chat.options.temperature=0.7
```

#### 3. The Injection
You can immediately inject the configured builder into your services.

```java
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class QuickChatController {

    private final ChatClient chatClient;

    // Spring automatically provides the pre-configured ChatClient.Builder
    public QuickChatController(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    @GetMapping("/joke")
    public String tellJoke() {
        return chatClient.prompt()
                .user("Tell me a joke about Java developers.")
                .call()
                .content();
    }
}
```

### Swapping Providers via Configuration

The true magic of Spring AI's auto-configuration combined with its unified abstractions is portability. 

If your company decides to move away from OpenAI to Anthropic's Claude due to privacy concerns, you **do not change your Java code**.

You simply:
1. Swap the dependency in `pom.xml` to `spring-ai-anthropic-spring-boot-starter`.
2. Update `application.properties`:
```properties
spring.ai.anthropic.api-key=${ANTHROPIC_API_KEY}
spring.ai.anthropic.chat.options.model=claude-3-5-sonnet-20240620
```

The auto-configuration detects Anthropic, creates an Anthropic-specific `ChatClient.Builder`, and injects it into your `QuickChatController`. The business logic remains entirely untouched.

### Vector Store Auto-Configuration

The same logic applies to Vector Databases. If you include the pgvector starter and provide the database credentials, Spring Boot will automatically configure a `VectorStore` bean that handles embedding and database insertion transparently.

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pgvector-store-spring-boot-starter</artifactId>
</dependency>
```

By relying heavily on auto-configuration, Spring AI drastically reduces the boilerplate required to build modern AI applications, allowing developers to focus on prompt engineering and business logic rather than wiring HTTP clients.
