# RAG architecture in Java

Retrieval-Augmented Generation (RAG) is a powerful pattern used to enhance the capabilities of Large Language Models (LLMs) by providing them with external, domain-specific knowledge that they were not originally trained on. In enterprise Java applications, implementing RAG involves combining document processing, vector embeddings, similarity search, and prompt engineering.

### Why RAG?
LLMs suffer from "hallucinations" (inventing facts) and have a knowledge cutoff date. If you ask an LLM about your company's internal HR policies, it will guess or fail. RAG solves this by first *Retrieving* relevant documents from a database and then *Augmenting* the prompt sent to the LLM to *Generate* a correct answer based on those documents.

### The RAG Pipeline in Java

A standard RAG pipeline consists of two main phases: the Ingestion Phase and the Retrieval/Generation Phase.

#### 1. Ingestion Phase
This phase happens asynchronously, usually when documents are uploaded or updated.
1. **Document Loading:** Reading PDFs, text files, or database records. In Spring AI, you might use `PdfDocumentReader` or `TextReader`.
2. **Splitting/Chunking:** Large documents exceed the context window of LLMs. They must be split into smaller, meaningful chunks (e.g., 500 tokens). `TokenTextSplitter` is commonly used.
3. **Embedding:** Converting the text chunks into dense vector representations (arrays of floats) using an Embedding Model (like `text-embedding-ada-002` or a local Ollama model).
4. **Vector Storage:** Saving the vectors and their original text metadata into a Vector Database (like pgvector, Milvus, or Qdrant).

#### 2. Retrieval & Generation Phase
This phase happens synchronously when a user makes a query.
1. **Query Embedding:** The user's text question is converted into a vector using the same Embedding Model.
2. **Similarity Search:** The application queries the Vector Database to find the most "similar" text chunks (usually using Cosine Similarity).
3. **Prompt Augmentation:** The retrieved text chunks are injected into a predefined Prompt Template alongside the user's original question.
4. **LLM Generation:** The augmented prompt is sent to the LLM, which generates a comprehensive answer based solely on the provided context.

### Java Code Example (Spring AI)

Here is a simplified example of the Retrieval and Generation phase using Spring AI.

```java
import org.springframework.ai.chat.ChatClient;
import org.springframework.ai.chat.messages.UserMessage;
import org.springframework.ai.chat.prompt.Prompt;
import org.springframework.ai.chat.prompt.SystemPromptTemplate;
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;
import java.util.Map;
import java.util.stream.Collectors;

@RestController
public class RagController {

    private final ChatClient chatClient;
    private final VectorStore vectorStore;

    // The system prompt template instructing the LLM on how to behave.
    private final String RAG_PROMPT_TEMPLATE = """
            You are a helpful assistant. Use the following pieces of context to answer the user's question.
            If you don't know the answer based on the context, just say that you don't know, don't try to make up an answer.
            
            Context:
            {context}
            """;

    public RagController(ChatClient chatClient, VectorStore vectorStore) {
        this.chatClient = chatClient;
        this.vectorStore = vectorStore;
    }

    @GetMapping("/ask")
    public String askQuestion(@RequestParam String question) {
        // 1. Retrieve similar documents from the Vector Store
        List<Document> similarDocuments = vectorStore.similaritySearch(
            SearchRequest.query(question).withTopK(3)
        );

        // Extract the content from the retrieved documents
        String contextContent = similarDocuments.stream()
                .map(Document::getContent)
                .collect(Collectors.joining("\n\n"));

        // 2. Augment the System Prompt with the retrieved context
        SystemPromptTemplate systemPromptTemplate = new SystemPromptTemplate(RAG_PROMPT_TEMPLATE);
        var systemMessage = systemPromptTemplate.createMessage(Map.of("context", contextContent));
        
        // Create the user message
        UserMessage userMessage = new UserMessage(question);

        // 3. Generate the answer by calling the LLM with the augmented prompt
        Prompt prompt = new Prompt(List.of(systemMessage, userMessage));
        return chatClient.call(prompt).getResult().getOutput().getContent();
    }
}
```

This architecture ensures that enterprise applications remain grounded in verifiable facts and can dynamically update their knowledge base without requiring expensive model retraining.
