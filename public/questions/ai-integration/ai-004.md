# Vector DBs (Qdrant/Milvus) integration in Java

While leveraging existing relational databases like PostgreSQL (with pgvector) is convenient, high-performance enterprise AI applications often require purpose-built **Vector Databases** to handle massive scale, ultra-low latency, and advanced filtering capabilities. In the Java ecosystem, integrating robust vector databases like **Qdrant** and **Milvus** is essential for building scalable Retrieval-Augmented Generation (RAG) and semantic search pipelines.

### Why Dedicated Vector Databases?

Dedicated vector databases are optimized from the ground up for high-dimensional vector similarity search. 
1. **Performance at Scale:** They employ advanced indexing algorithms (like HNSW, DiskANN) optimized for billions of vectors.
2. **Distributed Architecture:** Native support for sharding, replication, and high availability.
3. **Advanced Filtering:** They support complex pre-filtering and post-filtering based on metadata payloads, ensuring searches are both accurate and contextually constrained.

### Integrating Qdrant

Qdrant is a popular, Rust-based vector search engine known for its speed and powerful payload filtering system.

#### Spring AI Qdrant Integration

Spring AI provides auto-configuration for Qdrant, making integration seamless.

**1. Dependencies:**
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-qdrant-store-spring-boot-starter</artifactId>
</dependency>
```

**2. Configuration (application.properties):**
```properties
spring.ai.vectorstore.qdrant.host=localhost
spring.ai.vectorstore.qdrant.port=6334
spring.ai.vectorstore.qdrant.collection-name=my_collection
spring.ai.vectorstore.qdrant.api-key=your-api-key
```

**3. Usage:**
```java
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.ai.vectorstore.filter.FilterExpressionBuilder;
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class QdrantService {
    private final VectorStore vectorStore;

    public QdrantService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    public List<Document> searchWithFilter(String query, String category) {
        var b = new FilterExpressionBuilder();
        
        // Qdrant allows powerful metadata filtering alongside similarity search
        return vectorStore.similaritySearch(
            SearchRequest.query(query)
                .withTopK(5)
                .withFilterExpression(b.eq("category", category).build())
        );
    }
}
```

### Integrating Milvus

Milvus is an open-source, highly scalable vector database designed for massive data processing and enterprise-level production deployments. It is widely used in enterprise environments.

#### Spring AI Milvus Integration

Similar to Qdrant, Milvus is supported natively by Spring AI.

**1. Dependencies:**
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-milvus-store-spring-boot-starter</artifactId>
</dependency>
```

**2. Configuration (application.properties):**
```properties
spring.ai.vectorstore.milvus.client.host=localhost
spring.ai.vectorstore.milvus.client.port=19530
spring.ai.vectorstore.milvus.collection-name=enterprise_docs
spring.ai.vectorstore.milvus.embedding-dimension=1536
spring.ai.vectorstore.milvus.index-type=HNSW
spring.ai.vectorstore.milvus.metric-type=L2
```

**3. Usage:**
The programmatic usage for Milvus via Spring AI is identical to Qdrant, thanks to the `VectorStore` abstraction.

```java
import org.springframework.ai.vectorstore.VectorStore;
// ... (imports omitted for brevity)

@Service
public class MilvusService {
    private final VectorStore vectorStore;

    public MilvusService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    public void ingestData(List<Document> documents) {
        // The framework handles connecting to Milvus via gRPC and storing the vectors
        vectorStore.add(documents);
    }
}
```

### Choosing Between Them

- **Qdrant** is highly recommended when you need a lightweight, fast, and easy-to-deploy solution with exceptional metadata filtering capabilities. It is written in Rust and operates very efficiently.
- **Milvus** is often chosen for massive-scale enterprise architectures that require deep distributed computing features, complex sharding, and integration with large data ecosystems.

By leveraging Spring AI's unified `VectorStore` interface, Java developers can easily switch between Qdrant, Milvus, and other providers without rewriting their core application logic, ensuring future-proof architectural decisions.
