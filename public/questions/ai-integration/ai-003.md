# pgvector in PostgreSQL with Spring Data

As Retrieval-Augmented Generation (RAG) applications scale, managing vector embeddings effectively becomes crucial. While dedicated vector databases like Pinecone, Milvus, and Qdrant exist, many enterprise Java applications prefer leveraging their existing PostgreSQL infrastructure. This is where **pgvector**, an open-source extension for PostgreSQL, shines. Combining pgvector with **Spring Data** and **Spring AI** provides a robust, transactional, and unified persistence layer.

### What is pgvector?

`pgvector` adds native support for vector data types and vector similarity search to PostgreSQL. It allows you to store AI-generated embeddings (arrays of floating-point numbers) in standard tables alongside your relational data. It supports indexing techniques like IVFFlat and HNSW for fast, approximate nearest neighbor (ANN) searches using metrics like Cosine Similarity, Euclidean Distance, and Inner Product.

### Advantages of using pgvector in Spring Apps

1. **Reduced Operational Complexity:** You don't need to spin up and maintain a separate, specialized vector database.
2. **ACID Compliance:** Vector operations are fully transactional. You can insert a document's metadata and its embedding in a single transaction.
3. **Relational Joins:** You can filter vector searches based on complex relational conditions (e.g., "Find similar documents, but only those authored by user X and created in the last 7 days").
4. **Cost Efficiency:** Utilizes existing database resources.

### Integration with Spring AI

Spring AI provides an out-of-the-box `VectorStore` implementation for pgvector. This allows you to interact with pgvector using standard Spring AI abstractions without writing raw SQL.

#### 1. Database Setup
First, ensure the pgvector extension is installed in your PostgreSQL instance:
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

#### 2. Dependencies
Add the necessary Spring AI and PostgreSQL JDBC dependencies to your `pom.xml` or `build.gradle`.

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pgvector-store-spring-boot-starter</artifactId>
</dependency>
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
</dependency>
```

#### 3. Configuration
Configure the connection and vector store properties in your `application.properties`:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/mydb
spring.datasource.username=user
spring.datasource.password=pass

# Spring AI pgvector properties
spring.ai.vectorstore.pgvector.dimensions=1536 # For OpenAI text-embedding-ada-002
spring.ai.vectorstore.pgvector.index-type=HNSW
spring.ai.vectorstore.pgvector.distance-type=COSINE_DISTANCE
spring.ai.vectorstore.pgvector.initialize-schema=true
```

#### 4. Usage in Code
Once configured, Spring Boot auto-configures a `VectorStore` bean that you can inject into your services.

```java
import org.springframework.ai.document.Document;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.Map;

@Service
public class DocumentService {

    private final VectorStore vectorStore;

    public DocumentService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    public void saveDocument(String text, String author) {
        // Creating a document with metadata
        Document doc = new Document(text, Map.of("author", author));
        
        // The VectorStore automatically calls the EmbeddingClient to generate vectors,
        // then saves the document and vector to PostgreSQL via pgvector.
        vectorStore.add(List.of(doc));
    }

    public List<Document> searchSimilarDocuments(String query) {
        // Perform a similarity search based on Cosine Distance
        return vectorStore.similaritySearch(
            SearchRequest.query(query).withTopK(5)
        );
    }
}
```

### Advanced Usage with Spring Data JDBC/JPA

If you need more control than the Spring AI `VectorStore` provides (e.g., complex joins), you can map the vector column using Hibernate 6, which supports custom data types.

```java
import jakarta.persistence.Entity;
import jakarta.persistence.Id;
import org.hibernate.annotations.JdbcTypeCode;
import org.hibernate.type.SqlTypes;
import java.util.UUID;

@Entity
public class Article {
    @Id
    private UUID id;
    
    private String content;
    
    // Mapping the vector column
    @JdbcTypeCode(SqlTypes.VECTOR)
    private float[] embedding;
    
    // Getters and setters
}
```

By leveraging pgvector, Java developers can build powerful, production-ready RAG applications while maintaining the reliability and simplicity of standard PostgreSQL architectures.
