# AI Agents & ReAct in Java
Explain the concept of an AI Agent and the ReAct (Reasoning and Acting) prompting strategy. How can you implement an Agent in Java using frameworks like LangChain4j?

---ANSWER---

An **AI Agent** is a system where a Large Language Model (LLM) acts as the reasoning engine to determine a sequence of actions required to achieve a complex goal. Unlike a simple RAG pipeline where the flow is hardcoded (Embed -> Search -> Prompt -> Generate), an Agent is autonomous. Given a high-level task, a set of tools (functions), and memory, the Agent decides *which* tools to call, in *what order*, and interprets the results to determine its next step until the goal is met.

The most common paradigm for building Agents is the **ReAct (Reasoning and Acting)** prompting strategy. 
ReAct solves a core problem: LLMs are prone to hallucinating actions or getting stuck in loops. ReAct forces the LLM to interleave internal reasoning ("Thought") with explicit actions ("Action").

A typical ReAct loop looks like this:
1.  **Thought:** The LLM analyzes the current state and what it needs to do next. (e.g., "I need to find the user's order ID first, then look up the shipping status.")
2.  **Action:** The LLM requests to use a specific tool. (e.g., `findOrderId(userId="123")`)
3.  **Observation:** The system executes the tool and provides the result back to the LLM. (e.g., "Order ID is 999")
4.  The loop repeats until the LLM determines in its "Thought" phase that it has enough information to provide the final answer to the user.

**Implementing Agents in Java (LangChain4j)**

LangChain4j excels at creating Agents through its `AiServices` abstraction, making it heavily declarative.

1.  **Define the Interface:** You define a simple Java interface representing the Agent's entry point.
2.  **Provide Tools:** You create Java classes with methods annotated with `@Tool`, representing the actions the Agent can take (e.g., `DatabaseTool`, `CalculatorTool`, `WebSearchTool`).
3.  **Provide Memory:** Agents need Chat Memory to remember the context of the conversation and the sequence of steps they have already taken.
4.  **Assemble with AiServices:** You use the `AiServices.builder()` to glue the ChatModel, the Interface, the Tools, and the Memory together.

Under the hood, LangChain4j intercepts the method call to your interface. It manages the ReAct loop: sending the prompt, receiving the tool call, executing the Java method, feeding the observation back to the LLM, and repeating this until the LLM generates a final text response. This complex orchestration is entirely hidden from the developer, making Agent creation in Java surprisingly simple.

### Examples
```java
// 1. Define Tools
class ECommerceTools {
    @Tool("Get the order ID for a user name")
    public String getOrderId(String userName) { return "ORD-555"; }
    
    @Tool("Get shipping status for an order ID")
    public String getShippingStatus(String orderId) { return "In Transit to Warehouse"; }
}

// 2. Define Agent Interface
interface CustomerServiceAgent {
    @SystemMessage("You are a helpful customer support agent. Use tools to find information.")
    String handleInquiry(String userMessage);
}

// 3. Build and Run the Agent
public class AgentDemo {
    public static void main(String[] args) {
        ChatLanguageModel model = OpenAiChatModel.builder().apiKey("demo").build();
        
        CustomerServiceAgent agent = AiServices.builder(CustomerServiceAgent.class)
            .chatLanguageModel(model)
            .chatMemory(MessageWindowChatMemory.withMaxMessages(10))
            .tools(new ECommerceTools())
            .build();
            
        // The Agent will internally:
        // 1. Think: I need the order ID for Alice.
        // 2. Act: call getOrderId("Alice") -> gets "ORD-555"
        // 3. Think: Now I need status for ORD-555.
        // 4. Act: call getShippingStatus("ORD-555") -> gets "In Transit"
        // 5. Think: I have the answer.
        // 6. Return final response.
        String response = agent.handleInquiry("Where is the order for Alice?");
        System.out.println(response); 
    }
}
```

### Life Analogy
Think of an AI Agent like a Master Chef (the LLM) in a kitchen. You give the Chef a high-level goal: "Make a beef stew." 
The Chef doesn't do everything by magic. They use **ReAct**:
*   **Thought:** "I need beef, but it's frozen."
*   **Action:** Uses the "Microwave Defrost Tool".
*   **Observation:** Beef is thawed.
*   **Thought:** "Now I need to chop it."
*   **Action:** Uses the "Knife Tool".
*   **Observation:** Beef is chopped.
The Chef (LLM) autonomously decides the sequence of tools required to achieve the goal based on the observations at each step.

### Key Points
- An AI Agent uses an LLM to autonomously orchestrate a sequence of actions to achieve a goal.
- ReAct (Reasoning and Acting) is a prompting strategy that forces the LLM to explicitly state its thoughts before taking an action, reducing hallucinations.
- LangChain4j's `AiServices` provides a powerful, declarative way to build complex Agents in Java by simply combining an interface, `@Tool` methods, and Chat Memory.
