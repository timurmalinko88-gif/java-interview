# AI Agents & ReAct in Java

As Large Language Models (LLMs) evolve from simple text generators to complex reasoning engines, the paradigm of **AI Agents** has become central. An AI Agent is a system that uses an LLM as its "brain" to autonomously plan, use tools, and execute a sequence of actions to achieve a specific goal. 

A foundational architecture for building these agents is **ReAct** (Reasoning and Acting). 

### The ReAct Framework

The ReAct pattern structures how an LLM processes complex tasks. Instead of trying to guess the final answer immediately, the agent iteratively cycles through three steps:
1. **Thought (Reasoning):** The LLM analyzes the user request, the current state, and decides what to do next.
2. **Action (Acting):** The LLM decides to use a specific tool (Function Calling) with specific parameters.
3. **Observation:** The system executes the tool and feeds the result back to the LLM.

This loop continues until the LLM's "Thought" concludes that it has enough information to provide the final answer.

### Building Agents in Java

While Spring AI provides excellent building blocks, **LangChain4j** is currently the dominant framework in the Java ecosystem for building autonomous, stateful agents because it was designed around these concepts from day one.

#### Implementation using LangChain4j

In LangChain4j, an Agent is typically implemented using the `@AiService` declarative interface combined with `ChatMemory` and `@Tool` annotations.

**1. Defining the Tools (The Agent's "Hands")**
The agent needs tools to interact with the world.
```java
import dev.langchain4j.agent.tool.Tool;
import org.springframework.stereotype.Component;

@Component
public class ServerManagementTools {

    @Tool("Gets the current CPU usage of a specified server IP.")
    public double checkCpuUsage(String ipAddress) {
        // Mock implementation
        return 85.5; 
    }

    @Tool("Restarts a specified server by IP address.")
    public String restartServer(String ipAddress) {
        return "Server " + ipAddress + " successfully restarted.";
    }
}
```

**2. Defining the AI Service (The Agent's "Brain")**
We define an interface and give it a Persona (System Prompt) that instructs it to act autonomously.

```java
import dev.langchain4j.service.SystemMessage;
import dev.langchain4j.service.spring.AiService;

@AiService
public interface DevOpsAgent {

    @SystemMessage({
        "You are an autonomous DevOps engineering agent.",
        "Your goal is to ensure system stability.",
        "If a user reports an issue, investigate using your tools.",
        "If CPU usage is above 80%, you are authorized to restart the server.",
        "Always explain your reasoning before taking an action."
    })
    String handleRequest(String userRequest);
}
```

**3. Execution (The ReAct Loop in Action)**
When a user makes a request, LangChain4j handles the intricate ReAct loop under the hood.

```java
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AgentController {

    private final DevOpsAgent agent;

    public AgentController(DevOpsAgent agent) {
        this.agent = agent;
    }

    @PostMapping("/ops")
    public String executeDevOpsTask(@RequestBody String request) {
        /*
         * Example User Request: "The server at 192.168.1.50 is acting slow."
         * 
         * Under the hood ReAct sequence:
         * 1. Thought: User says server is slow. I should check CPU usage first.
         * 2. Action: checkCpuUsage("192.168.1.50")
         * 3. Observation: Returns 85.5
         * 4. Thought: CPU is 85.5%, which is > 80%. My instructions authorize a restart.
         * 5. Action: restartServer("192.168.1.50")
         * 6. Observation: "Server 192.168.1.50 successfully restarted."
         * 7. Thought: Issue resolved. I will inform the user.
         */
        return agent.handleRequest(request); 
        // Returns: "I checked the CPU usage and found it at 85.5%. As this is over the 80% threshold, I have restarted the server to restore stability."
    }
}
```

### Challenges with Agents

1. **Infinite Loops:** An agent can get stuck in a loop of failing tool calls. Timeouts and max-iteration limits are essential.
2. **Non-Determinism:** Because LLMs are probabilistic, the agent might solve a problem in 3 steps today and 5 steps tomorrow.
3. **Security:** Autonomous agents are highly susceptible to prompt injection. Never give an agent destructive tools without strict validation or human-in-the-loop oversight.
