# Local Embeddings models (Ollama)

When building AI applications, especially Retrieval-Augmented Generation (RAG) systems, security, privacy, and cost are major enterprise concerns. Sending highly sensitive corporate documents to third-party APIs (like OpenAI or Anthropic) for embedding generation is often a compliance violation. The solution is running **Local Embedding Models** using tools like **Ollama**.

### What are Embeddings?
Embeddings are high-dimensional vector representations of text. They capture the semantic meaning of sentences, allowing algorithms to determine how "similar" two pieces of text are. Generating these vectors requires an AI model.

### Why Use Local Embeddings via Ollama?

Ollama is a lightweight, extensible framework that allows you to run open-source Large Language Models (LLMs) and embedding models directly on your local machine or internal servers.

1. **Data Privacy:** Your text never leaves your infrastructure. 
2. **Zero API Costs:** Embedding generation can be highly intensive during the ingestion phase of RAG. Running locally eliminates per-token API charges.
3. **Latency:** Local models can reduce network overhead, leading to faster vector generation.
4. **Customization:** You can select specialized models (e.g., models fine-tuned for legal or medical texts) like `nomic-embed-text` or `mxbai-embed-large`.

### Integration with Spring AI

Spring AI provides excellent, out-of-the-box support for Ollama, allowing you to seamlessly replace cloud-based embedding clients with local ones.

#### 1. Setup Ollama
First, install Ollama on your machine and pull an embedding model. A popular and highly efficient embedding model is `mxbai-embed-large` or `nomic-embed-text`.
```bash
ollama run nomic-embed-text
```

#### 2. Maven/Gradle Dependencies
Include the Spring AI Ollama starter in your Java project.
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-ollama-spring-boot-starter</artifactId>
</dependency>
```

#### 3. Configuration
Configure your `application.properties` to point the `EmbeddingClient` to your local Ollama instance and specify the model.

```properties
# Base URL for Ollama (default is localhost:11434)
spring.ai.ollama.base-url=http://localhost:11434

# Configure the embedding model specifically
spring.ai.ollama.embedding.options.model=nomic-embed-text
```

#### 4. Java Implementation

Once configured, Spring Boot will automatically inject an `OllamaEmbeddingModel` implementation of the `EmbeddingModel` interface.

```java
import org.springframework.ai.embedding.EmbeddingModel;
import org.springframework.ai.embedding.EmbeddingResponse;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

@RestController
public class EmbeddingController {

    private final EmbeddingModel embeddingModel;

    public EmbeddingController(EmbeddingModel embeddingModel) {
        this.embeddingModel = embeddingModel;
    }

    @GetMapping("/embed")
    public List<Double> generateEmbedding(@RequestParam String text) {
        // Call the local Ollama model to generate vectors
        EmbeddingResponse response = embeddingModel.embedForResponse(List.of(text));
        
        // Extract and return the float array representing the text
        return response.getResult().getOutput();
    }
}
```

### Best Practices

1. **Hardware Acceleration:** While Ollama runs on CPU, it heavily benefits from GPUs. For production embedding pipelines, deploying Ollama on servers with Nvidia GPUs or Apple Silicon will drastically reduce embedding times.
2. **Dimension Matching:** Remember that local embedding models often produce vectors of different dimensions than OpenAI (e.g., `nomic-embed-text` outputs 768 dimensions, while OpenAI's `text-embedding-ada-002` outputs 1536). You must ensure your Vector Database table is initialized with the correct dimensions matching your local model.
3. **Dedicated Embedding Models:** Ensure you use models designed specifically for embedding (like `nomic-embed-text`), not general chat models (like `llama3`), as chat models are inefficient and inaccurate for vector generation.
