# Structured Output (BeanOutputConverter)
When integrating an LLM into a Java application, you rarely want a raw String response. How do you force an LLM to return structured data (like JSON) that maps directly to a Java Record or POJO?

---ANSWER---

One of the biggest challenges when integrating Large Language Models (LLMs) into traditional software systems is that LLMs inherently output unstructured natural language (String), whereas applications expect structured data (JSON, XML, Java Objects). If an LLM returns a conversational response when your application expects a JSON payload, parsing will fail, and the application will break.

To solve this, AI frameworks provide mechanisms to enforce **Structured Output**. In the Java ecosystem, this is typically achieved through Output Converters, such as Spring AI's `BeanOutputConverter`.

The process involves two coordinated steps:
1.  **Prompt Instruction (Format Engineering):** The framework automatically injects explicit instructions into the system prompt telling the LLM exactly how to format its response. It analyzes your Java class (e.g., a Record or POJO) and generates a JSON Schema. The prompt is modified to say: "Return the response strictly as JSON that adheres to the following schema..."
2.  **Deserialization:** Once the LLM returns the string (which should now be valid JSON), the framework intercepts it and uses a standard JSON library (like Jackson) to deserialize that string directly into the expected Java object.

**How it works in Spring AI:**
Spring AI provides the `BeanOutputConverter<T>`. 
*   You instantiate it with the target Java `class` or `ParameterizedTypeReference` (useful for `List<T>`).
*   You call `converter.getFormat()` to get the generated format instructions (the JSON schema) and append it to your user prompt.
*   After calling the `ChatClient`, you pass the raw string response to `converter.convert(rawString)`. This parses the JSON and returns your strongly typed Java object.

Modern models (like GPT-4o) have native support for "JSON Mode" or "Structured Outputs", making them extremely reliable at adhering to the requested schema. The Java frameworks usually leverage these native API features under the hood when available, falling back to prompt engineering for older models.

### Examples
```java
// Spring AI Example
record MovieRecommendation(String title, int year, String reason) {}

@Service
public class MovieService {
    private final ChatClient chatClient;

    public MovieService(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    public MovieRecommendation recommendMovie(String genre) {
        // 1. Create the converter for the specific Record
        BeanOutputConverter<MovieRecommendation> converter = 
            new BeanOutputConverter<>(MovieRecommendation.class);

        // 2. The converter automatically generates prompt instructions based on the Record's fields
        String formatInstructions = converter.getFormat();

        // 3. Append instructions to the prompt
        String prompt = "Recommend a good {genre} movie. " + formatInstructions;

        // 4. Call the LLM
        String rawContent = chatClient.prompt()
                .user(u -> u.text(prompt).param("genre", genre))
                .call()
                .content();

        // 5. Convert the raw JSON string back into the Java Record
        return converter.convert(rawContent);
    }
}
```

### Life Analogy
Asking an LLM a question without structured output is like asking a contractor for an invoice and getting a hand-written letter that says, "Hey, great working with you! The pipes cost 50 bucks and labor was 200. Cheers!" It's hard for your accounting software to read.
Using `BeanOutputConverter` is like giving the contractor a strict fill-in-the-blank PDF form. You explicitly tell them (via prompt instructions) to only fill in the boxes for "Item", "Cost", and "Total". When they hand it back, your software can instantly process the data.

### Key Points
- Applications need structured data (JSON/Objects), not conversational text.
- `BeanOutputConverter` (in Spring AI) bridges this gap by automatically generating JSON Schemas from Java classes.
- It injects these schemas into the prompt instructions, forcing the LLM to output valid JSON.
- It then automatically deserializes the LLM's JSON string response back into the strongly-typed Java Record or POJO.
