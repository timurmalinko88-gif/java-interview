# Question

Review the following `ShoppingCart` class. It represents a user's cart in an e-commerce application. Point out any design flaws or bugs, focusing on Object-Oriented principles.

```java
import java.util.ArrayList;
import java.util.List;

public class ShoppingCart {
    private final List<String> items;
    private double totalDiscount;

    public ShoppingCart() {
        this.items = new ArrayList<>();
        this.totalDiscount = 0.0;
    }

    public void addItem(String item) {
        this.items.add(item);
    }

    public List<String> getItems() {
        return this.items;
    }

    public double getTotalDiscount() {
        return totalDiscount;
    }

    public void setTotalDiscount(double totalDiscount) {
        if (totalDiscount < 0) {
            throw new IllegalArgumentException("Discount cannot be negative");
        }
        this.totalDiscount = totalDiscount;
    }
}
```

## Answer

### Life Analogy
Returning the raw internal list is like giving a customer the keys to your store's cash register so they can read the total, but trusting them not to take out any money. 

### Bugs and Issues
1. **Leaked Encapsulation (Mutable State)**: `getItems()` returns the internal `List` reference. A caller can do `cart.getItems().clear()` or `cart.getItems().add("Free TV")`, bypassing any validation logic the `ShoppingCart` might have.
   - *Fix*: Return an unmodifiable view: `return Collections.unmodifiableList(items);` or return a copy `return new ArrayList<>(items);`.
2. **Anemic Domain Model**: The `ShoppingCart` has a setter for `totalDiscount`. This allows any external class to arbitrarily overwrite the discount. The cart should ideally control its own business rules (e.g., `applyPromoCode(String code)`).

### Key Points for a Correct Answer
- Must identify that `getItems()` breaks encapsulation by leaking a mutable reference.
- Must propose returning an unmodifiable list or a defensive copy.
- Should point out that setters for business values (`setTotalDiscount`) violate the OOP principle of objects managing their own state and invariants.
