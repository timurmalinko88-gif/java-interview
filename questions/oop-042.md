# Subclassing and Static Method Hiding

### Question
Review the following Java 21 code snippet. The developer wanted to apply polymorphism to calculate discounts for different customer types. Identify the bugs and explain why the output is not what the developer expects.

```java
import java.util.List;
import java.util.ArrayList;

class Customer {
    public static double getDiscount() {
        return 0.0;
    }
    
    public void printDiscount() {
        System.out.println("Discount: " + getDiscount());
    }
}

class VIPCustomer extends Customer {
    // Attempting to override the discount logic
    public static double getDiscount() {
        return 0.20;
    }
}

public class DiscountService {
    public static void main(String[] args) {
        Customer regular = new Customer();
        Customer vip = new VIPCustomer();
        
        System.out.println("Regular:");
        regular.printDiscount();
        
        System.out.println("VIP:");
        vip.printDiscount(); // Developer expects 0.20 to be printed
    }
}
```

### Analogy
Static methods belong to the *building* (the Class), while instance methods belong to the *tenant* (the Object). You can't ask the tenant about a building rule and expect a personalized answer—they'll just read you the building's rulebook.

### Answer Structure

#### 1. The Core Issue: Method Hiding vs Overriding
The code compiles perfectly, but it contains a fundamental misunderstanding of polymorphism in Java. Static methods are resolved at **compile-time** based on the reference type, not at **runtime** based on the actual object type. 

#### 2. The Bugs Identified
- **Bug 1: Static Method "Overriding" (Method Hiding).** 
  `VIPCustomer.getDiscount()` does not override `Customer.getDiscount()`; it *hides* it. Since `printDiscount()` is an instance method in `Customer`, the call to `getDiscount()` inside it is statically bound to `Customer.getDiscount()`.
- **Bug 2: Output failure.** 
  Calling `vip.printDiscount()` will print `Discount: 0.0`, not `0.20`, because `printDiscount()` executes in the context of the `Customer` class where `getDiscount()` evaluates to `0.0`.

#### 3. The Fix
Remove the `static` keyword from `getDiscount()` in both classes. This makes the methods instance methods, allowing standard dynamic method dispatch (runtime polymorphism) to execute the correct overriding method.

```java
class Customer {
    public double getDiscount() { return 0.0; } // Removed static
    public void printDiscount() {
        System.out.println("Discount: " + this.getDiscount());
    }
}

class VIPCustomer extends Customer {
    @Override
    public double getDiscount() { return 0.20; } // Removed static
}
```

- [ ] Explain that static method calls are resolved at compile-time based on the reference type.
- [ ] Point out that `vip.printDiscount()` will output `0.0`.
- [ ] Propose the correct fix: removing the `static` keyword to leverage runtime polymorphism.
