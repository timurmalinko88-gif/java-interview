Спроектируйте сервис сокращения URL-адресов (наподобие bit.ly).
Опишите основные компоненты и как вы будете генерировать короткие ссылки.

---ANSWER---

**Основные компоненты:**
1. **API Gateway / Load Balancer:** Балансирует нагрузку от пользователей.
2. **Shortener Service:** Генерирует короткий хеш.
3. **Redirect Service:** По хешу ищет длинный URL в кеше или БД, и отдает `HTTP 301/302 Redirect`.
4. **Database (RDBMS/NoSQL):** Хранит пары `(short_url -> long_url)`.
5. **Cache (Redis):** Кеширует частые запросы для быстрого редиректа.

**Генерация ссылок (Кодирование Base62):**
Алфавит (A-Z, a-z, 0-9) содержит 62 символа. Строка из 7 таких символов дает ~3.5 трлн комбинаций.
- Берется уникальный числовой ID (например, сгенерированный через Snowflake алгоритм или счетчик из БД).
- Этот числовой ID переводится из 10-чной системы счисления в 62-чную.
- Полученная строка сохраняется как `short_url`. Это полностью исключает коллизии.
