How does the Java Virtual Machine determine that an object in memory is no longer required and can safely be reclaimed? What are Garbage Collection Roots (GC Roots), and what are at least three distinct types of them?

---ANSWER---

### 1. Tracing vs. Reference Counting

Historically, some languages used Reference Counting to track object lifespans (releasing memory when an object's reference counter drops to zero). The HotSpot JVM does not use this approach because it fails to handle circular references (e.g., Object A points to Object B, and Object B points back to Object A, but neither is referenced by anything else).

Instead, modern JVMs utilize Reachability Analysis (or reference tracing). It builds a directed graph of references starting from a set of trusted, inherently active starting points called GC Roots.

> `[ GC Root ] ---> [ Object A ] ---> [ Object B ] (Reachable - Retained)`
> 
> `                 [ Object C ] <---> [ Object D ] (Unreachable - Reclaimed)`

An object is considered "live" (retained) if there is an active reference path connecting it to any GC Root. If an object is unreachable, it is eligible for garbage collection, even if it contains internal circular references.

### 2. Definition of a GC Root

A GC Root is an object that is implicitly assumed to be active, reachable, and alive by definition, serving as the starting anchor points for the GC tracing algorithm.

### 3. Key Categories of GC Roots

* **Thread Stack Variables:** Any local variables, reference arrays, and method parameters residing on the stack frames of currently running threads.
* **Static Class Fields:** Static variables declared in classes. These variables remain alive as long as their respective ClassLoader is loaded and active.
* **JNI Local & Global References:** Java objects accessed by native C or C++ code compiled via the Java Native Interface (JNI).
* **JVM Internal References:** System-level references held by the JVM, such as the system ClassLoaders, active Thread groups, and system-level exception handlers.
