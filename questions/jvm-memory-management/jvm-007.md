# Code Review: Cache built with String keys — spot the memory and correctness bugs
A junior developer wrote a simple in-memory cache for HTTP response bodies. Review the code below, identify **all bugs**, explain their impact, and provide the corrected version.

```java
import java.util.HashMap;

public class ResponseCache {

    // Bug area 1
    private static HashMap<String, byte[]> cache = new HashMap<>();

    public static void store(String url, byte[] body) {
        // Bug area 2
        if (!cache.containsKey(url)) {
            cache.put(new String(url), body);  // line A
        }
    }

    public static byte[] retrieve(String url) {
        return cache.get(url);
    }

    public static void evict(String url) {
        cache.remove(url);
    }

    // Bug area 3
    public static int size() {
        return cache.size();
    }

    public static void main(String[] args) {
        String endpoint = new String("https://api.example.com/data"); // line B
        ResponseCache.store(endpoint, "payload".getBytes());

        String lookup = new String("https://api.example.com/data");  // line C
        byte[] result = ResponseCache.retrieve(lookup);
        // Bug area 4
        if (result == null) {
            System.out.println("Cache miss — but we just stored it!");
        } else {
            System.out.println("Hit: " + result.length + " bytes");
        }
    }
}
```

---

## Life Analogy

Think of a **library catalogue**: the catalogue (cache) stores books by title cards. Bug 1 is like having a single catalogue open to everyone in the building simultaneously with no locking — two librarians could corrupt the same card. Bug 2 is like gluing a photocopy of the title card to the shelf instead of using the standard card — wasteful and confusing. The cache itself works fine in this small `main()` because `HashMap.get()` uses `.equals()` internally, so the lookup still hits — but the unnecessary `new String()` at line A wastes memory for no reason.

---

---ANSWER---

1. **`new String(url)` as a map key (line A) is wasteful but not a correctness bug here**  
   `HashMap` uses `.equals()` and `.hashCode()`, so lookups still work. However, this creates an unnecessary heap duplicate and should be replaced with `url` directly (or the string is already interned).

2. **`static HashMap` is not thread-safe**  
   Concurrent `store()` / `evict()` calls can corrupt the map. Should use `ConcurrentHashMap` or explicit synchronization — this is the most critical production bug.

3. **`containsKey` + `put` is a non-atomic check-then-act**  
   Two threads can both pass `containsKey`, then both call `put`, potentially overwriting each other. `putIfAbsent()` or `computeIfAbsent()` fixes this atomically with `ConcurrentHashMap`.

4. **`byte[]` values are mutable and stored by reference**  
   The cache stores a direct reference to the caller's array. If the caller mutates the array after storing, cached data is silently corrupted. A defensive copy (`Arrays.copyOf(body, body.length)`) is needed.

5. **`new String(endpoint)` at line B and line C are unnecessary**  
   Using string literals directly or the original reference is cheaper. The code works correctly because `HashMap.get()` uses `.equals()`, but it misleads readers into thinking reference equality matters.

---

## Bug Summary

| # | Location | Type | Severity |
|---|---|---|---|
| 1 | `static HashMap` | Thread safety | Critical |
| 2 | `containsKey` + `put` | Race condition | High |
| 3 | `new String(url)` as key (line A) | Memory waste | Low |
| 4 | Mutable `byte[]` stored by reference | Data corruption | Medium |

## Corrected Version

```java
import java.util.Arrays;
import java.util.concurrent.ConcurrentHashMap;

public class ResponseCache {

    private static final ConcurrentHashMap<String, byte[]> cache = new ConcurrentHashMap<>();

    public static void store(String url, byte[] body) {
        // atomic: only inserts if key is absent
        cache.putIfAbsent(url, Arrays.copyOf(body, body.length));
    }

    public static byte[] retrieve(String url) {
        byte[] data = cache.get(url);
        // return defensive copy so callers can't corrupt cached state
        return data != null ? Arrays.copyOf(data, data.length) : null;
    }

    public static void evict(String url) {
        cache.remove(url);
    }

    public static int size() {
        return cache.size();
    }

    public static void main(String[] args) {
        String endpoint = "https://api.example.com/data";
        ResponseCache.store(endpoint, "payload".getBytes());

        byte[] result = ResponseCache.retrieve(endpoint);
        if (result == null) {
            System.out.println("Cache miss");
        } else {
            System.out.println("Hit: " + result.length + " bytes");
        }
    }
}
```
