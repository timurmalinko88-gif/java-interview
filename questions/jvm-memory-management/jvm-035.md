Describe the ClassLoader Delegation Model in Java. What is the fundamental difference between a `ClassNotFoundException` and a `NoClassDefFoundError`? Provide realistic scenarios that trigger each of them.

---ANSWER---

### 1. The ClassLoader Delegation Model

When a ClassLoader receives a request to load a class, it does not attempt to locate it immediately. It delegates the request to its Parent ClassLoader first. This delegation bubbles up to the top of the hierarchy:

* **Bootstrap ClassLoader:** Loads core JDK classes (e.g., packages inside `java.base` such as `java.lang.String`). It is written in native C/C++ and returns `null` if queried in Java.
* **Platform ClassLoader:** (formerly Extension ClassLoader) Loads platform-specific extensions.
* **Application (System) ClassLoader:** Loads classes from your application's classpath (your code and external dependencies).

Only if the parent hierarchy fails to find the class does the current ClassLoader attempt to load the class itself. This prevents duplicate classes and security breaches (e.g., an application trying to hijack `java.lang.String`).

### 2. ClassNotFoundException vs. NoClassDefFoundError

| Metric | `ClassNotFoundException` | `NoClassDefFoundError` |
| :--- | :--- | :--- |
| **Type** | Checked Exception (inherits from `Exception`). | Linkage Error (inherits from `Error`). |
| **Trigger Time** | Dynamic runtime execution. | Class Linking / Static Initialization. |
| **Cause** | Occurs when an application tries to load a class dynamically by its name string (e.g., via reflection) but cannot find its `.class` file on the classpath. | Occurs when the class was present during compile-time, but is missing, corrupted, or failed to initialize during runtime. |
| **Typical Code** | `Class.forName("com.mysql.cj.jdbc.Driver")` without having the driver dependency added. | `new HelperClass()` where `HelperClass.class` is absent from the deployment JAR, or `HelperClass` threw an exception inside its `static {}` block. |

### Mandatory Key Points

* **Delegation Flow:** Describe how loading requests bubble up to the parent ClassLoader (Bootstrap) before resolving downwards.
* **Checked vs. Linkage Error:** Distinguish that `ClassNotFoundException` is a checked Exception, whereas `NoClassDefFoundError` is a fatal system Error.
* **Reflection vs. Linkage:** Explain that `ClassNotFoundException` occurs during dynamic lookups (reflection), while `NoClassDefFoundError` occurs when compile-time dependencies go missing at runtime.
* **Static Initializer Failure:** Note that a failure in a static block (`static {}`) can trigger a `NoClassDefFoundError` upon subsequent attempts to use that class.

## Analogy

* **ClassLoader Delegation** is like asking a lawyer's assistant for a document. The assistant first delegates the search up to their Senior Partner. The Partner delegates it up to the Managing Director. Only if the Managing Director cannot find it in the executive vault (Bootstrap) does the search move down to the regional archive (Platform), and finally back down to the local file cabinet (Application).
* **ClassNotFoundException** is like typing a misspelled book title into a library database. The program tells you: "No book by this name exists in our system."
* **NoClassDefFoundError** is like pulling a book off the shelf that you already read yesterday (compiled fine), but finding out that someone has physically ripped all the chapters out of the binding since then.
