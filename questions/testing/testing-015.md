# BDD vs TDD

What is Behavior-Driven Development (BDD)?

---ANSWER---

BDD is an extension of TDD that focuses on the *behavior* of the system from the user's perspective. It uses domain-specific language (like Gherkin: Given, When, Then) so that non-technical stakeholders (business analysts, QA) can read and write the test scenarios. Frameworks like Cucumber are often used.

### Life Analogy
TDD is an engineer writing structural blueprints for a house. BDD is the interior designer and homeowner agreeing on a story: "Given I walk in, When I flip the switch, Then the lights turn on."

### Key Points
- Focuses on user behavior and specifications.
- Uses Given/When/Then format.
- Bridges the gap between technical and non-technical teams.
