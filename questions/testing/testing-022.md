# Test Coverage

What is Test Coverage and is 100% coverage necessary?

---ANSWER---

Test coverage measures the percentage of your code executed by tests. While high coverage (e.g., 80%) is good, 100% is rarely necessary. Chasing 100% often leads to testing trivial code (like getters/setters) which adds maintenance overhead without real value.

### Life Analogy
It's like installing security cameras. Covering all main rooms and doors is smart, but putting a camera inside every drawer is paranoid and expensive.

### Key Points
- Measures code execution during tests.
- High coverage != bug-free code.
