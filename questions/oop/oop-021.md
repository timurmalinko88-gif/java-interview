# Understanding Polymorphism and Method Dispatch
In Java, how does the JVM distinguish between **Method Overloading** and **Method Overriding** under the hood? Explain why one is considered "static" and the other "dynamic," and how this impacts performance and behavior.

---ANSWER---

Polymorphism in Java manifests in two primary ways, handled fundamentally differently by the JVM.

**1. Method Overloading (Compile-Time Polymorphism)**
- **Binding:** Static (Early Binding).
- **JVM Instruction:** Resolved by the compiler using `invokestatic` or `invokespecial` (and sometimes `invokevirtual` but strictly bound to the signature).
- **Mechanism:** The compiler looks at the reference type and the arguments passed. It selects the exact method signature at compile time.

**2. Method Overriding (Run-Time Polymorphism)**
- **Binding:** Dynamic (Late Binding).
- **JVM Instruction:** Uses the `invokevirtual` instruction (or `invokeinterface`).
- **Mechanism:** The JVM looks at the *actual object type* in memory at runtime (via the vtable/virtual method table), not the reference type. It then invokes the most specific overridden method.

### Key Mandatory Points for a Correct Answer
1. **Static vs. Dynamic Binding:** Must clearly state overloading is resolved at compile-time (static binding) and overriding at run-time (dynamic binding).
2. **Reference vs. Object Type:** Must explain that overloading depends on the reference type of arguments, whereas overriding depends on the actual object type in the heap.
3. **VTable (Virtual Table):** A strong answer mentions that the JVM uses a vtable to resolve overridden methods dynamically.
4. **Performance Impact:** Dynamic dispatch carries a tiny performance overhead due to vtable lookup, whereas static dispatch is marginally faster and easily inlined by the JIT compiler.

- **Overloading** is like ordering a "Burger". You can order a "Burger (beef)" or a "Burger (chicken)". The waiter knows *exactly* which one you mean at the time you order based on your words (Compile-time).
- **Overriding** is like asking the "Chef of the Day" to make their "Signature Dish". You don't know exactly how it will be prepared until the specific chef on duty actually cooks it (Run-time).
