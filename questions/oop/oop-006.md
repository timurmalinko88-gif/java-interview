# Question: The Fragile Base Class

Many tutorials start by teaching inheritance using `Animal` -> `Dog`. However, in modern Java development, we often prefer Composition over Inheritance. 
Can you explain *why* composition is generally preferred, and describe a specific scenario where using inheritance can actually break encapsulation (often called the "Fragile Base Class" problem)?

---ANSWER---

- **Tight Coupling:** Inheritance creates the tightest form of coupling in Java. The subclass depends on the implementation details of the parent.
- **Breaking Encapsulation:** If a superclass method's internal implementation changes (e.g., a method starts calling another overridden method internally), it can unexpectedly break subclass behavior without changing the method signature.
- **Flexibility:** Composition allows behavior to be changed at runtime (using interfaces/Strategy pattern), whereas inheritance is locked in at compile time.
- **Single Inheritance Limit:** Java only allows extending one class, consuming your one chance at inheritance, whereas a class can compose as many objects as it needs.

### 1. The Core Difference
Inheritance implies an "IS-A" relationship, whereas composition implies a "HAS-A" relationship. While inheritance promotes code reuse, it forces a rigid hierarchy. Composition achieves code reuse by injecting dependencies, keeping classes focused and decoupled.

### 2. The Fragile Base Class Problem
This occurs when a base class is modified in a seemingly safe way, but it inadvertently breaks subclasses. 
For example, if a subclass overrides `add()` and `addAll()` methods to count elements, and later the base class author modifies `addAll()` to internally call `add()`, the subclass will double-count elements. The subclass relied on the *implementation details* of the parent, meaning encapsulation was broken.

### 3. When to Actually Use Inheritance
Inheritance is appropriate when:
- The "IS-A" relationship is mathematically strict.
- You control both the parent and child classes (they evolve together).
- You are building an internal framework hierarchy where polymorphism relies strictly on inherited state.

- **Inheritance** is like moving into a fully furnished apartment. If the landlord replaces the comfy sofa with a wooden bench, you're stuck with it because you are tightly bound to their choices. 
- **Composition** is like renting an unfurnished apartment and buying your own furniture. You use the apartment's structure, but you completely control the pieces inside it.
