# Question: Static vs Dynamic Binding in Java

In Java, what is the difference between static (early) binding and dynamic (late) binding? Provide a concrete example of how the Java compiler and JVM handle both.

---ANSWER---

1. Mentions that static binding happens at compile-time based on reference type.
2. Mentions that dynamic binding happens at runtime based on the actual object type.
3. Correctly identifies method overloading as static binding.
4. Correctly identifies method overriding as dynamic binding.
5. Notes that `private`, `static`, and `final` methods use static binding.

### Life Analogy
Think of static binding like a **pre-booked train ticket**: your seat and destination are fixed before the journey starts. Dynamic binding is like an **Uber ride**: the exact route and driver are determined on the fly based on current conditions when you request the ride.

### Key Points
*   **Static Binding (Early Binding):** Happens at compile-time. The compiler determines which method to call based on the reference variable's type. It applies to `private`, `static`, and `final` methods, as well as overloaded methods.
*   **Dynamic Binding (Late Binding):** Happens at runtime. The JVM determines which method to call based on the actual object instance, not the reference type. This is the core mechanism of method overriding (polymorphism).
*   **Performance:** Static binding is slightly faster because the method call is resolved during compilation. Dynamic binding requires a virtual table (vtable) lookup at runtime.
