# The Fragile Base Class
"Favor composition over inheritance" is a common OOP mantra. Explain the "Fragile Base Class" problem that inheritance can cause, and describe how composition avoids this issue using a real-world software example (e.g., a notification system or billing calculator).

---ANSWER---

### Life Analogy
Inheritance is like buying a house with all the furniture bolted to the floor. If you want to change the sofa, you have to tear up the living room (fragile). Composition is like renting an unfurnished apartment and buying your own furniture; you can swap out the sofa anytime without affecting the building structure.

### Key Points
- **Fragile Base Class:** Changes in a superclass can unexpectedly break subclasses. Because subclasses depend on the internal implementation details of the parent, tight coupling occurs. 
- **Composition:** Assembling complex behavior by combining simpler, independent objects (often injecting them via interfaces) rather than inheriting them.
- **Example:** Instead of an `EmailNotifier` inheriting from `BaseNotifier`, a `NotificationService` *has a* `MessageSender` interface. We can inject an `EmailSender` or `SMSSender` without altering the core service.

### Mandatory Evaluation Points
1. Defines the Fragile Base Class problem (tight coupling, parent changes break children).
2. Explains that inheritance breaks encapsulation (subclasses know parent internals).
3. Demonstrates composition using an interface/dependency injection.
4. Identifies flexibility at runtime (Strategy pattern) as a benefit of composition.
